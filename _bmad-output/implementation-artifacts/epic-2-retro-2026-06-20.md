## What Went Well

### 1. Tests integrated from the start (Ian's highlight)
Tests built from Story 1.2 onward meant zero regression risk throughout the packaging work. When src/ improvements from Epic 1 slipped into the Story 2.1 working tree, the 74/74 green suite gave confidence to commit them separately without risk. The test suite actively invited completing coverage gaps rather than ignoring them.

### 2. Debug Log documented during development
Technical gotchas were recorded in-story as they occurred:
- `asar: false` → immediate silent exit (Electron 42 ASAR fuse)
- `asarUnpack` → ASAR integrity failure (exit code 255)
- Icon generation via PowerShell requiring absolute paths

This practice prevents institutional knowledge loss. Note: the Bitdefender entry needs correction (see Action Items).

### 3. Windows build end-to-end verified
The `npm run package` → install → launch → verify flow was completed on Windows 11. The installed app matches all visual and functional acceptance criteria.

### 4. Code review caught two real issues
- Source maps packaged inside the ASAR (`files: out/**` without exclusion) → fixed with `!**/*.map`
- Fragile test selector (`getByText('00:00:00')`) → fixed with `data-testid="timer-display"`

### 5. WIP band as intentional placeholder
The heart rate band architecture reserves space without commitment. Epic 3 (if it happens) has a clear integration point. No hidden debt.

## Challenges

### 1. Electron 42 undocumented behaviors (3 surprises)
- **`asar: false`**: Caused immediate exit with no error message. Root cause: Electron 42 enables the `onlyLoadAppFromAsar` fuse by default. Diagnostic time was significant.
- **`asarUnpack`**: ASAR integrity failure on exit. Same fuse family. Also undocumented for this version combination.
- **`asarUnpack` + code signing**: Would resolve the fuse issue but adds signing infrastructure complexity.

**Lesson:** When upgrading Electron to a major version, review Electron Fuses release notes before any packaging work.

### 2. "Always on top" manual testing was laborious — with a misleading diagnosis
Testing the always-on-top toggle proved difficult. The Debug Log attributes the issue to Bitdefender (nothing was actually quarantined or blocked). The real cause: a video game running in exclusive fullscreen on the primary screen interfered with window stacking during tests.

**Clarification from Ian:** In real usage, the app is on the secondary screen alongside Firefox, Chrome, Slack, and VS Code. The always-on-top feature works correctly in this configuration. The limitation (fullscreen exclusive game on the same screen) does not impact Ian's actual use case.

**Lesson:** Document test *conditions* as precisely as test *results*. "Always on top works" is insufficient. "Always on top works on secondary screen with standard windowed applications — not tested against exclusive fullscreen on the same screen" is actionable.

### 3. Scope drift: src/ modifications outside Story 2.1
Improvements to Epic 1 code (`App.tsx` optional chaining, `TimerWidget.tsx` tick cap, test refactors) accumulated in the working tree during Story 2.1 work despite the story explicitly forbidding `src/` changes. Detected at code review and committed separately.

**Lesson:** When implementing infrastructure stories that say "do not touch src/", stage only the infrastructure files before marking ready-for-review.

### 4. macOS build not verifiable from Windows
Cross-compilation (Windows → macOS) is not supported by electron-builder. `icon.icns` generation requires macOS. This was anticipated and documented, but it means Epic 2's delivery is technically incomplete until a macOS machine is available.

---

## Key Insights

1. **Tests as safety net enable confident scope decisions** — the suite made the "separate commit" decision for src/ changes straightforward rather than risky.
2. **Debug Log is valuable but requires accuracy** — a misleading entry (Bitdefender) wastes future debugging time. Cause must be verified before documenting.
3. **Environment conditions belong in acceptance criteria** — "Always on top" tests need to specify the window environment to be reproducible.
4. **Electron Fuses are a silent breaking change vector** — undocumented by default, high impact on packaging behavior.
